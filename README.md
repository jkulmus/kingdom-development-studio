# Overview

As a software development student, I am continuing to build a personal software portfolio that demonstrates my growth in web development, cloud computing, authentication, and database management. This project is Kingdom Development Studio, a cloud-based web application designed to organize and manage construction projects for a themed entertainment destination.

The software demonstrates several core JavaScript and cloud application concepts including:

- Cloud database integration using Firebase Firestore
- Firebase Authentication
- Create, Read, Update, and Delete (CRUD) operations
- Responsive web interface
- Modular JavaScript architecture
- Administrator login and authorization
- Dynamic dashboard statistics
- Search, filtering, and sorting
- Firestore Security Rules
- Interactive project management interface

The application allows users to browse construction projects across multiple kingdoms while authenticated administrators can securely manage project records stored in Cloud Firestore. Project information is dynamically loaded from the database and displayed through an interactive dashboard.

The purpose of my project is to strengthen my understanding of cloud-based application development, Firebase services, modular JavaScript organization, authentication, database security, and responsive user interface design while creating software around a project that I am passionate about.

# How to Run the Program

1. Clone or download the repository.
2. Open the project folder in Visual Studio Code.
3. Create a Firebase project.
4. Enable Firebase Authentication using Email/Password.
5. Create a Cloud Firestore database.
6. Update the Firebase configuration in `firebaseConfig.js`.
7. Open the project using Live Server.
8. Sign in using an administrator account to access project management features.

# Learning Strategies

One strategy that worked well during this project was building the application one feature at a time. I first created the basic dashboard interface before gradually adding searching, filtering, dashboard statistics, authentication, Firestore integration, and CRUD functionality. Testing each feature before moving on made troubleshooting much easier.

Another strategy that helped was reorganizing the project into separate JavaScript modules. Separating controllers, services, user interface components, and utility functions made the application much easier to maintain as it continued to grow.

One strategy I plan to improve in future projects is spending more time planning the application architecture before beginning implementation. Although I reorganized the project successfully, planning the folder structure earlier would have reduced the amount of refactoring required later.

# Challenges and Improvements

One of the biggest challenges during this project was learning how Firebase Authentication, Cloud Firestore, and Firestore Security Rules all work together. Understanding how to securely protect the database while allowing authenticated users to perform CRUD operations required significant research and experimentation.

Another challenge was preventing the project from becoming too large too quickly. Because the application is based on a much larger long-term idea, I frequently wanted to continue adding new features before completing existing ones. I overcame this by focusing on finishing one feature at a time before moving on to the next.
